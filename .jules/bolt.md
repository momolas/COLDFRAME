## 2025-10-24 - CoreLocation Updates Thrashing SwiftUI Render Loop
**Learning:** In iOS/SwiftUI, `CLLocationManager` without `headingFilter` or `distanceFilter` fires continuous, microscopic updates. When these are dispatched to an `@Observable` class marked with `@MainActor` (like `QiblaManager`), it forces the main thread to handle excessive property mutations. This causes the SwiftUI views observing those properties (e.g., compass rotation) to re-render dozens of times per second, leading to UI stutter and heavy battery drain, even if the user isn't physically moving.
**Action:** Always throttle `CLLocationManager` updates by setting `distanceFilter` (e.g., 100 meters, or depending on use-case) and `headingFilter` (e.g., 1 degree) upon initialization, especially when bridging legacy delegate callbacks to modern `@MainActor` state.
